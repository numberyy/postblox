# R6 — TUI polish closure plan

## Status

R6 is the current phase per `CLAUDE.md`: TUI polish — write-through actions,
command bar, themes. R3c landed the read-side TUI on the daemon socket; R6
closes the loop by making the TUI a fully usable mail client without leaving
the terminal. This document tracks the remaining slices needed to call R6
done. Each slice is independently shippable per the anti-bloat rules.

## Decisions

| Topic | Decision |
|-------|----------|
| Drafts autosave | Save on Esc/exit and explicit `:w` / `Ctrl-S`. No timer-based autosave. |
| Move target UX | Command-bar `:move <folder>` first; `m` opens command bar pre-filled with `:move `. |
| Forward attachments | Re-fetch from IMAP on demand; cache only what's already in DB; surface "attachment unavailable offline" if fetch fails. |
| Search scope default | All-accounts; `:search --account <name>` to scope. |
| Themes | `postblox.toml [tui] theme = "dark"` for default; `:theme <name>` runtime override (in-memory, not persisted). |
| Notifications | Bottom-of-screen transient toasts, 3s default, stack of 3 max, dismiss with `x`. |
| Status pane IDLE indicator | Per-account icon: `●` IDLE, `~` polling, `!` last-error, `…` syncing. |

## Slices

### Slice 1 — Write-through message actions

**Scope.** Archive, delete, move, mark read/unread, flag from the message list
and details pane. Optimistic local update; reconcile against `mail.*` hub
events.

**Daemon ops touched.** `messages.archive`, `messages.delete`, `messages.move`,
`messages.mark_read`, `messages.flag` (whichever already exist; if a verb is
missing, add it as part of this slice).

**TUI surface.** Keys: `e` archive, `d` delete with confirmation, `m` move
(`:move ` prefilled in command bar), `u` toggle read, `*` toggle flag.

**Tests.** Unit tests for action dispatch + reconcile; integration test
asserting optimistic UI rolls back on op error event.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 2 — Notifications + status pane signals

**Scope.** Subscribe to `mail.new`, `account.synced`, and account-error events.
Surface as bottom-of-screen toasts (queue cap 3, 3s default, dismiss with `x`).
Status pane gains per-account state icon.

**Daemon ops touched.** Existing subscription/event stream; no new ops.

**TUI surface.** Toast widget; status pane icons `●` IDLE, `~` polling, `!`
last-error, `…` syncing.

**Tests.** Toast queue cap test (4th notification evicts oldest); status icon
state-machine test.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 3 — Command bar parity

**Scope.** Add commands: `:compose`, `:reply`, `:reply-all`, `:forward`,
`:archive`, `:delete`, `:move <folder>`, `:goto <folder>`, `:account <name>`,
`:search [--account <name>] <query>`, `:theme <name>`. `:` triggers
autocompletion. Each command MUST call the same handler as its keybinding —
no duplicated logic.

**Daemon ops touched.** Same as the keybindings; pure dispatch wiring.

**TUI surface.** Command-bar input + completion popup.

**Tests.** Parser tests per command (positional, flags, escaping); routing
test that `:archive` and `e` invoke the same handler.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 4 — Theme polish

**Scope.** Light, dark, high-contrast palettes. `[tui] theme` config default.
Runtime `:theme <name>` switch (in-memory only). Central palette table
consumed by the render layer — no scattered ad-hoc colors.

**Daemon ops touched.** None.

**TUI surface.** Palette module; render call sites switched to palette
lookups.

**Tests.** Palette resolution test (config default, missing theme falls back,
runtime switch overrides).

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 5 — Search UI

**Scope.** `/` quick-search and `:search`. Results pane backed by existing
`search.query` op. Enter on a result jumps to the message and switches
folder/account context. Default scope is all-accounts.

**Daemon ops touched.** `search.query` (no new op).

**TUI surface.** Search input + results pane; scope flag rendering.

**Tests.** Result-jump context-switch test; scope flag parser test.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 6 — Compose attachments (outgoing)

**Scope.** Composer attachments panel. Add by path; remove; show running
size + count. Server-side 25 MB cap (per `CLAUDE.md` bounds) surfaced as a
toast/error when exceeded.

**Daemon ops touched.** Whatever the composer/draft path already uses for
attachments; surface size-cap rejection.

**TUI surface.** Attachments panel inside composer.

**Tests.** Add/remove path edits; cap-exceeded error path.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 7 — Reply / Reply-all / Forward

**Scope.** Correct `In-Reply-To` and `References` header construction. Subject
normalization (`Re:` / `Fwd:` collapse). Reply-all dedup of the account's own
primary email. Quoted body via the existing `mail/` reply extractor. Forward
re-fetches attachments via IMAP on demand; surfaces an offline fallback
message if the fetch fails.

**Daemon ops touched.** Compose/draft creation ops; on-demand attachment
fetch path.

**TUI surface.** Reply / reply-all / forward keybindings + command-bar verbs
land in the composer pre-filled.

**Tests.** Header construction unit tests (single message, threaded message,
missing `Message-ID`); subject normalization; reply-all dedup against each
account alias; forward offline fallback.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 8 — Drafts list / reopen / edit / delete

**Scope.** Drafts view (filter on Drafts folder by role). Open a draft into
the composer with full state restore. `:w` saves. Delete with confirmation.
Save-on-exit only — no timed autosave.

**Daemon ops touched.** Draft CRUD (existing).

**TUI surface.** Drafts list + composer reopen path.

**Tests.** Round-trip test: compose → save → reopen restores body, headers,
attachments.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 9 — Attachment preview scroll/select (optional polish)

**Scope.** Scroll, line select, `y` to copy text-preview content to the
clipboard. UI-only; no daemon changes.

**Daemon ops touched.** None.

**TUI surface.** Attachment preview pane.

**Tests.** Scroll bounds; clipboard yank target.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

### Slice 10 — Manual QA pass via `tuistory`

**Scope.** Drive the panes end-to-end via the `tuistory` skill. Capture
findings as follow-up issues unless trivial enough to fix in-line. Closes R6.

**Daemon ops touched.** None directly; bug fixes scoped per finding.

**TUI surface.** Whole app.

**Tests.** Manual; any fixes carry their own unit/integration tests.

**Validators.** `cargo fmt --check`, `cargo test`,
`cargo clippy --all-targets -- -D warnings`.

## Cross-cutting hygiene

- **H1 — Perf baseline.** Capture cold-start, idle memory, IPC ping p50, DB
  read p50 against the `CLAUDE.md` performance table. Land a tiny harness
  that prints these so regressions are visible.
- **H2 — `cargo deny check` CI gate.** Wire into the CI matrix alongside
  fmt/clippy/test. Required to merge.
- **H3 — Dead-code audit.** Run `cargo machete`,
  `cargo +nightly udeps`, and clippy `dead_code` lints. Delete what no live
  path or test references.

## Anti-bloat reminders

- Every PR must be usable on its own — no trait-setup-for-next-PR slices.
- No abstractions before the third concrete use; write the second handler
  duplicated rather than extracting a trait early.
- Delete code that isn't tested — untested code is a liability.
- Comments explain WHY; if a comment is restating WHAT the code does,
  simplify the code instead.
- AI-authored code gets compressed: halve the lines where possible after
  every session.
- Dependencies earn their place — prefer well-known crates; one-sentence
  justification or it doesn't land.
